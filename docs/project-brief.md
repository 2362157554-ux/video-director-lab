# Project Brief

## Goal

Build an experimental foundation for a non-technical video creation workflow.

A user should eventually be able to provide rough ideas, scripts, PDFs, screenshots, references, or media assets. The system should help turn those inputs into a structured video plan, choose suitable rendering paths, generate previews, inspect quality, and iterate before final delivery.

## Early Principles

1. Do not treat MP4 export as proof of quality.
2. Keep planning, rendering, and quality review as separate stages.
3. Treat Remotion and HyperFrames as complementary tools, not alternatives.
4. Treat MotionLayer as a possible visual/motion vocabulary, not a render engine by itself.
5. Keep provenance for generated outputs so every video can be traced back to its source plan, renderer, and quality report.
